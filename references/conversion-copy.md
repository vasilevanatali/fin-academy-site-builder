# Conversion And Copy

## Copy Rules

1. Write in direct Russian, without marketing fog.
2. Lead with the real offer, not a vague inspiration headline.
3. Use finance/business terminology naturally: выручка, ДДС, P&L, EBITDA, маржа, валовая прибыль, операционная прибыль.
4. Separate facts from assumptions. If a number is not provided, write `данных нет` in the brief and do not place it on the page.
5. Do not promise outcomes the product cannot guarantee.
6. Do not hide the price on paid mini-products unless the user explicitly asks.
7. Do not use "волшебная таблетка", passive income framing, fake scarcity, or exaggerated transformation claims.

## Hero Formula

Hero should answer in 5 seconds:

- what this is;
- for whom;
- what result or job it helps with;
- when/price/access if relevant;
- what to click next.

Good patterns:

- `[Product category] для [audience]: [specific job/result]`
- `[Tool/topic] для начинающих: за один вечер [practical outcome]`
- `[Professional path]: [a clear route] without vague hype`

Add one support line with practical detail, then CTA.

## Proof Blocks

Use only verified proof:

- real student/customer count;
- real screenshots/reviews;
- real author credentials;
- real artifacts: prompt pack, workbook, checklist, dashboard, financial model, lesson preview;
- real event/lesson dates.

If proof is visually weak but important, put it in a calm trust strip rather than oversized hero text.

## Tariffs

Use cards, not dense tables, unless the product genuinely needs comparison. For mobile, use tabs/cards/accordion.

Each tariff card should include:

- name;
- price;
- access period;
- who it is best for;
- included modules/features;
- clear CTA;
- payment note: card, SBP, invoice for legal entity, installment if confirmed.

Never mark "Выбор большинства" unless confirmed.

## FAQ

Include objections specific to the offer:

- "Подойдёт ли новичку?"
- "Когда открывается доступ?"
- "Можно ли оплатить от юрлица?"
- "Чем это отличается от большого курса?"
- "Будут ли записи?"
- "Есть ли обратная связь?"
- "Что если не подойдёт?"

If the answer depends on policy, leave a placeholder and ask the user.

## CTA Placement

Place CTA:

- in hero;
- after program/benefits;
- near tariff block;
- in final section;
- as mobile sticky after first screen.

CTA labels must be concrete:

- `Зарегистрироваться на эфир`
- `Забрать запись за 990 ₽`
- `Выбрать тариф`
- `Получить доступ`

Avoid vague labels like `Подробнее` as primary CTA.

## Public Promo Links

For public skill demos or repository description, use public links only:

- ChatGPT excursion: `https://chatgpt.fin-academy.pro`
- Claude excursion: `https://claude.fin-academy.pro`
- Codex excursion: `https://codex.fin-academy.pro`
- Telegram channel examples from [promo-links.md](promo-links.md)

Never include private student chats, invite links, order admin links, internal screenshots, exports, or credentials.
