# Design Directions

Use this reference at the start of a new page or redesign. Pick one primary direction and one secondary accent. Do not mix all directions.

## Selection Rule

Choose a direction based on:

- product type;
- audience seniority;
- available assets;
- price level;
- amount of proof;
- desired emotional tone.

State the chosen direction in the implementation plan before coding.

## Premium Fintech

Use for: finance products, probudget, financial director tools, executive lessons.

Feel:

- competent;
- calm;
- structured;
- high trust;
- not playful.

Layout:

- editorial hero with dashboard/workbook artifact;
- dense proof strip;
- structured program;
- dark tariff/final CTA anchor;
- precise tables/cards.

Visuals:

- document previews;
- spreadsheet/model screenshots;
- charts;
- restrained accent color;
- high contrast typography.

Avoid:

- cartoon AI visuals;
- bright neon;
- vague startup gradients.

## AI Lab / Neural Workshop

Use for: ChatGPT, Claude, Codex, AI tools, prompt lessons.

Feel:

- modern;
- practical;
- experimental but controlled;
- tool-first.

Layout:

- hero with route map or product screenshots;
- artifact tabs;
- before/after outputs;
- prompt preview;
- lesson/player preview;
- CTA near practical examples.

Visuals:

- real UI screenshots;
- code/prompt fragments;
- generated images only when tied to the lesson;
- subtle neural/grid motif.

Avoid:

- fake sci-fi;
- generic robot illustrations;
- unreadable dark screenshots.

## Editorial Expert

Use for: expert-led webinars, interviews, методички, thought-leadership pages.

Feel:

- intelligent;
- human;
- magazine-like;
- credible.

Layout:

- author/photo hero;
- pull quotes;
- agenda as editorial sections;
- proof via credentials and public work;
- FAQ as objections.

Visuals:

- speaker portrait;
- notes/doc fragments;
- calm typography;
- selective accent blocks.

Avoid:

- overdesigned SaaS look;
- hiding the expert behind abstract visuals.

## Product Demo

Use for: paid recordings, mini-courses, templates, dashboards, prompt packs.

Feel:

- tangible;
- useful;
- immediate;
- "I can see what I get".

Layout:

- artifact-led hero;
- preview gallery;
- included-items grid;
- before/after section;
- price/access block;
- FAQ about access and level.

Visuals:

- screenshots;
- workbook pages;
- dashboard/table previews;
- checklist fragments;
- lesson player mockup.

Avoid:

- selling abstract benefits without showing the product.

## Consulting / Executive

Use for: outsourcing, consulting, diagnostics, implementation offers.

Feel:

- serious;
- operational;
- premium;
- responsibility-focused.

Layout:

- problem/job framing;
- diagnostic checklist;
- process timeline;
- deliverables;
- boundaries and expectations;
- application CTA.

Visuals:

- process maps;
- muted palette;
- precise diagrams;
- limited motion.

Avoid:

- course-style hype;
- too many small feature cards.

## Course Platform

Use for: larger programs, multi-module courses, training products.

Feel:

- organized;
- comprehensive;
- supportive;
- clear path.

Layout:

- path/roadmap hero;
- modules accordion;
- lesson previews;
- author/team block;
- tariffs;
- student/support FAQ.

Visuals:

- module cards;
- progress/path motifs;
- lesson thumbnails;
- support/community cues if confirmed.

Avoid:

- overwhelming all modules above the fold;
- feature lists without learning outputs.

## Direction Handoff

When choosing a direction, report:

- primary direction;
- why it fits;
- chosen layout moments;
- assets needed;
- risks if assets are missing.
