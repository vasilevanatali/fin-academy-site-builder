# Visual Swipe File

Use this as a pattern library. Do not copy external websites. Apply the mechanics to the current offer, content, and assets.

## Hero Patterns

### Artifact Hero

Best for: mini-course, recording, prompt pack, workbook, dashboard.

Structure:

- headline with product category and audience;
- support copy with practical job;
- CTA + price/access/date;
- large artifact mockup;
- 3 proof chips under hero.

Conversion job: make the product tangible.

### Route Map Hero

Best for: tool экскурсия, AI lesson, implementation workshop.

Structure:

- headline promises guided route;
- visual map of stops;
- each stop has tool/task/result;
- CTA to join/get access.

Conversion job: reduce fear of complex tools.

### Expert Hero

Best for: webinar, consulting, executive page.

Structure:

- speaker visual;
- short credential proof;
- agenda/outcomes;
- date/time/CTA.

Conversion job: establish authority fast.

## Proof Patterns

### Trust Strip

Use near top.

Content:

- verified numbers;
- public links;
- artifacts;
- author credentials;
- known projects.

If no proof, use context strip instead of fake stats.

### Claim + Evidence Pair

For every strong claim, show evidence next to it:

- screenshot;
- lesson fragment;
- quote;
- public example;
- model/checklist preview.

## Explanation Patterns

### Before / After

Use for: AI workflow, site redesign, finance process.

Structure:

- left: current manual/chaotic process;
- right: improved process after product;
- center or bottom: what changes.

### Job Tabs

Use for mixed audiences:

- финансист;
- собственник;
- эксперт;
- команда;
- новичок.

Each tab must change meaningful content, not just labels.

### Timeline / Route

Use for:

- program;
- implementation;
- live event agenda;
- consulting phases.

Mobile: vertical cards or accordion.

## Artifact Patterns

### Preview Gallery

Show 3-5 artifacts:

- prompt pack;
- workbook;
- dashboard;
- checklist;
- lesson player;
- site preview.

Each preview needs a caption: what it helps do.

### Annotated Screenshot

Use callouts to explain:

- where to click;
- what changed;
- what result is shown;
- what the student receives.

Keep private data blurred/replaced.

### Copyable Prompt Teaser

Use for AI products:

- show 5-10 lines of a useful prompt;
- include a "full pack inside" CTA;
- do not reveal paid material if it should stay private.

## Tariff Patterns

### Single Offer Block

Use for low-ticket or simple product:

- price;
- access term;
- included items;
- payment CTA;
- what happens after payment.

### Two-Tier Choice

Use when there are two clear jobs:

- basic access;
- access plus support/templates/review.

Avoid arbitrary feature padding.

### Executive Application

Use for consulting:

- not a price table;
- scope and deliverables;
- who it is for/not for;
- application CTA;
- next step after request.

## Interaction Patterns

Use when content benefits from state:

- artifact tabs;
- FAQ accordion;
- module accordion;
- before/after slider;
- tariff toggle;
- sticky side summary;
- progress/anchor nav.

Do not use interaction for critical content if it hides what the user must know before payment.

## Final CTA Patterns

### Summary CTA

Restate:

- product;
- for whom;
- price/access/date;
- what happens after click.

### Risk-Reducer CTA

Use when price or trust is barrier:

- what is included;
- payment method;
- access timing;
- support/contact link;
- FAQ anchor.

### Urgent CTA

Use only with real event/deadline:

- real date;
- real deadline;
- no fake counters.

## Pattern Quality Check

Before finalizing:

- every pattern has a conversion job;
- no repeated card grid dominates the page;
- artifacts are real or labeled placeholders;
- mobile version preserves the pattern's meaning;
- CTA remains visible after the pattern.
